# cursovyi-ptoect
AdAgencyCRM — це десктопний застосунок для автоматизації діяльності менеджера рекламного агентства, розроблений на платформі .NET 8.0 з використанням Windows Forms. Програма призначена для управління клієнтською базою, рекламним інвентарем (зокрема, LED-екранами), оформлення замовлень та формування звітності.

Основні можливості

Управління клієнтами: Ведення бази даних клієнтів (додавання, редагування, перегляд історії).
Управління послугами: Облік рекламних площин (LED-екрани, білборди, друк тощо) з детальними технічними характеристиками (розмір, локація, час роботи).
Автоматизація замовлень:
Створення замовлень з автоматичним розрахунком вартості.
Перевірка доступності ефірного часу на LED-екранах (запобігання "овербукінгу").
Розрахунок ПДВ та знижок.
Звітність: Формування звітів про продажі за період з можливістю фільтрації по менеджерах та статусах.
Експорт: Експорт звітів у формат CSV (сумісний з Excel).
Безпека: Система авторизації користувачів (Admin/Manager) з хешуванням паролів.

Технічний стек

Платформа: .NET 8.0 (Desktop)
Мова програмування: C#
Інтерфейс користувача (UI): Windows Forms
База даних: MySQL
ORM: Entity Framework Core 8.0
Провайдер БД: Pomelo.EntityFrameworkCore.MySql
Вимоги до системи
Для коректної роботи застосунку необхідні наступні компоненти:
Операційна система: Windows 10/11 (x64)
Середовище виконання: .NET Desktop Runtime 8.0
СУБД: MySQL Server 8.0 або новішої версії

Налаштування та запуск

1. Налаштування бази даних
Перед першим запуском переконайтеся, що у вас встановлено MySQL Server.
Відкрийте файл AdAgencyCRM/Data/AppDbContext.cs.

Знайдіть рядок підключення в методі OnConfiguring:
var connectionString = "server=localhost;user=root;password=ВАШ_ПАРОЛЬ;database=adagencydb";

Замініть user та password на ваші облікові дані MySQL.

2. Застосування міграцій
Застосунок використовує Code-First підхід. База даних буде створена автоматично при першому запуску, або ви можете створити її вручну через консоль диспетчера пакетів:
dotnet ef database update


3. Перший вхід
При першому створенні бази даних автоматично створюється адміністратор за замовчуванням:

Логін: admin
Пароль: (хеш пароля прописаний у AppDbContext.cs, за замовчуванням це admin)

Структура проєкту

Forms: Містить форми інтерфейсу користувача (MainForm, CreateOrderForm, LoginForm та ін.).
Models: Класи сутностей бази даних (Client, Order, Service, User та ін.).
Data: Контекст бази даних (AppDbContext).
Services: Бізнес-логіка (наприклад, LedAvailabilityService для перевірки вільного часу).
Utils: Допоміжні утиліти (MoneyCalculator, UserSession).
Reports: Логіка формування та експорту звітів.
